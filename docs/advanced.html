⚡ Advanced Features

Deep-dives into Pillar's most powerful features: the embedded task queue, database layer, DDD enforcement, the Smart Bridge, and production observability.

Background Task Queue #

Pillar ships with a persistent task queue backed by the Rust engine. No Redis. No Celery. No separate worker process. Tasks are written to SQLite WAL mode and survive server restarts.

Pillar Queue Architecture
🦀 PillarQueue (Rust)
SQLite WAL · uuid task IDs · chrono timestamps · rusqlite-bundled
↕ PyO3 FFI
🐍 TaskStorage + TaskWorker (Python)
serialise calls · asyncio poller · thread-pool executor · retry logic
💡 When to use Pillar Queue vs Redis

Start with the SQLite driver (default). Upgrade to Redis when you need: horizontal scaling across multiple processes, sub-millisecond enqueue latency, or millions of tasks per hour. Change one line in pillar.toml — your Python code stays identical.

@background_task Decorator #

pythonpython
from pillar import background_task

class EmailService:
    @background_task(retries=3)
    def send_welcome(self, to: str, name: str):
        """
        Calling self.send_welcome(email, name) does NOT run this code now.
        It serialises the call and returns a task_id string.
        The TaskWorker picks it up and runs it asynchronously.
        On failure it retries up to 3 times before marking as failed.
        """
        smtp.send(
            to=to,
            subject=f"Welcome, {name}!",
            body=render_template("welcome.html", name=name),
        )

    @background_task(retries=5)
    def send_invoice(self, order_id: int):
        order = order_repo.get(order_id)
        pdf   = pdf_generator.render(order)
        smtp.send_pdf(to=order["email"], pdf=pdf)

    @background_task(retries=0)
    def log_signup(self, user_id: int, ip: str):
        analytics.track("signup", user_id=user_id, ip=ip)

What happens when you call a queued function

  1. The call is serialised (function path + args + kwargs) into the SQLite queue
  2. The function returns a task ID string immediately — no blocking
  3. The background TaskWorker polls the queue and executes the task in a thread pool
  4. On exception, the task is retried up to retries times
  5. The queue persists to disk — tasks survive server restarts
decorator parameterspython
@background_task(
    retries=3,      # int  — retry on failure; 0 = no retry
    cron=None,      # str  — future: "*/5 * * * *" for scheduled tasks
)

Module-level Tasks #

core/tasks.pypython
import shutil
from pillar import background_task

@background_task(retries=1)
def purge_tmp_uploads(older_than_hours: int = 24):
    import time, os, pathlib
    cutoff = time.time() - older_than_hours * 3600
    for f in pathlib.Path("/tmp/uploads").glob("*"):
        if f.stat().st_mtime < cutoff:
            f.unlink(missing_ok=True)

@background_task(retries=2)
def sync_to_s3(local_path: str, s3_key: str):
    import boto3
    s3 = boto3.client("s3")
    s3.upload_file(local_path, "my-bucket", s3_key)

# Call anywhere — returns task_id
task_id = purge_tmp_uploads(older_than_hours=48)
print(f"Enqueued: {task_id}")

Testing Background Tasks #

tests/test_email.pypython
from unittest.mock import patch, MagicMock

def test_welcome_email_is_queued(email_service):
    """Assert the task is enqueued (not executed) during the request."""
    with patch.object(email_service, "send_welcome") as mock_queue:
        email_service.on_user_created(user_id=1, email="a@b.com")
        mock_queue.assert_called_once_with("a@b.com", "Alice")

def test_welcome_email_logic(email_service):
    """Test the actual email logic by bypassing the queue."""
    with patch("smtp.send") as mock_smtp:
        # Call _original_func directly — skips the queue entirely
        email_service.send_welcome._original_func(
            email_service, to="a@b.com", name="Alice"
        )
        mock_smtp.assert_called_once()
        call_kwargs = mock_smtp.call_args.kwargs
        assert call_kwargs["to"] == "a@b.com"
        assert "Welcome" in call_kwargs["subject"]
ℹ _original_func

Every @background_task-decorated function stores a reference to the original unwrapped function at func._original_func. Use this in tests to call the logic directly without going through the queue.

Database Layer #

Pillar provides a Database class with thread-local SQLite connections. It's registered as a framework singleton and injected anywhere you type-hint db: Database. WAL mode and foreign key enforcement are on by default.

Database API #

MethodReturnsUse for
db.query(sql, params)dict | NoneSingle row lookup
db.query_all(sql, params)list[dict]Multiple rows
db.execute(sql, params)int (rowcount)INSERT / UPDATE / DELETE
db.execute_returning(sql, params)dict | NoneWrite and return the row
db.last_insert_id()int | NoneRowid after INSERT
db.close()—Close thread's connection
pythonpython
from pillar.db import Database

class ArticleRepository:
    def __init__(self, db: Database):
        self.db = db
        self.db.execute("""
            CREATE TABLE IF NOT EXISTS articles (
                id         INTEGER PRIMARY KEY AUTOINCREMENT,
                title      TEXT    NOT NULL,
                body       TEXT    NOT NULL,
                author_id  INTEGER NOT NULL,
                published  INTEGER DEFAULT 0,
                created_at TEXT    DEFAULT (datetime('now'))
            )
        """)

    def get(self, article_id: int) -> dict | None:
        return self.db.query(
            "SELECT * FROM articles WHERE id = ?", (article_id,)
        )

    def list_published(self, limit: int = 20, offset: int = 0) -> list:
        return self.db.query_all(
            "SELECT * FROM articles WHERE published = 1 "
            "ORDER BY created_at DESC LIMIT ? OFFSET ?",
            (limit, offset)
        )

    def create(self, title: str, body: str, author_id: int) -> dict:
        self.db.execute(
            "INSERT INTO articles (title, body, author_id) VALUES (?,?,?)",
            (title, body, author_id)
        )
        return self.get(self.db.last_insert_id())

    def publish(self, article_id: int) -> int:
        return self.db.execute(
            "UPDATE articles SET published = 1 WHERE id = ?", (article_id,)
        )

Repository Patterns #

search + filter patternpython
def search(
    self,
    query:     str     = "",
    category:  str     = "",
    min_price: float   = 0,
    max_price: float   = float("inf"),
    limit:     int     = 20,
    offset:    int     = 0,
) -> list:
    conditions = ["1=1"]
    params: list = []

    if query:
        conditions.append("name LIKE ?")
        params.append(f"%{query}%")
    if category:
        conditions.append("category = ?")
        params.append(category)
    if min_price > 0:
        conditions.append("price >= ?")
        params.append(min_price)
    if max_price < float("inf"):
        conditions.append("price <= ?")
        params.append(max_price)

    params.extend([limit, offset])
    return self.db.query_all(
        f"SELECT * FROM products WHERE {' AND '.join(conditions)} "
        f"ORDER BY name LIMIT ? OFFSET ?",
        tuple(params)
    )

Transactions #

pythonpython
from pillar.db import Database

class OrderRepository:
    def __init__(self, db: Database): self.db = db

    def create_with_items(self, order_data: dict, items: list) -> dict:
        conn = self.db._conn()
        try:
            conn.execute("BEGIN")
            conn.execute(
                "INSERT INTO orders (user_id, total) VALUES (?,?)",
                (order_data["user_id"], order_data["total"])
            )
            order_id = conn.execute(
                "SELECT last_insert_rowid()"
            ).fetchone()[0]

            for item in items:
                conn.execute(
                    "INSERT INTO order_items (order_id, product_id, qty, price)"
                    " VALUES (?,?,?,?)",
                    (order_id, item["product_id"], item["qty"], item["price"])
                )
                conn.execute(
                    "UPDATE products SET stock = stock - ? WHERE id = ?",
                    (item["qty"], item["product_id"])
                )

            conn.execute("COMMIT")
            return self.get(order_id)
        except Exception:
            conn.execute("ROLLBACK")
            raise

DDD Layer Enforcer #

At startup, Pillar uses Python's ast module to parse every router.py, service.py, and repository.py inside domains/. If any file imports from a forbidden layer, ArchitectureViolationError is raised before the first request.

This check runs once at startup — zero runtime overhead.

Import Matrix #

repository.py
✓ schemas
✗ service
✗ router
service.py
✓ repository
✓ schemas
✗ router
router.py
✓ service
✓ schemas
✗ repository
ℹ Only domains/ is checked

Files in core/, tests/, or the project root are not subject to enforcement. If the domains/ directory does not exist, the enforcer is skipped entirely.

Violation Error Output #

startup error exampletext
pillar.exceptions.ArchitectureViolationError:

  File   : domains/orders/router.py
  Layer  : router.py
  Imports: 'domains.orders.repository'
  Problem: 'router' must not import from 'repository'.
  Allowed: ['schemas', 'service']

  Fix: move the logic that needs 'repository' into a
  layer that is allowed to use it.

The Smart Bridge #

Pillar's handler invocation layer (the Smart Bridge) inspects your function at call time using inspect.iscoroutinefunction():

  • async function → called directly on the event loop with await
  • sync function → dispatched to a thread pool via asyncio.run_in_executor(None, ...)

This means your sync database queries never block the event loop, and your async I/O calls never create unnecessary threads. You just write Python — the framework handles execution context.

Smart Bridge Patterns #

python — both workpython
# Pattern 1: Async handler + sync service + sync repository
# (most common — sync DB code runs in thread pool automatically)
@router.get("/{id}")
async def get_product(id: int, svc: ProductService):
    return svc.get(id)   # svc.get() is sync → thread pool

class ProductService:
    def get(self, id: int) -> dict:          # sync
        return self.repo.get(id)             # sync

class ProductRepository:
    def get(self, id: int) -> dict | None:   # sync
        return self.db.query("SELECT …", (id,))

# ---

# Pattern 2: Sync handler (entire chain runs in thread pool)
@router.get("/report")
def generate_report(svc: ReportService):    # sync handler
    return svc.build_heavy_report()          # runs in thread pool

# ---

# Pattern 3: Async handler + async service (LLM calls etc.)
@router.post("/chat")
async def chat(data: ChatRequest, svc: AIService):
    return await svc.stream_completion(data.prompt)  # async LLM call

class AIService:
    async def stream_completion(self, prompt: str) -> dict:
        response = await openai_client.chat(prompt)
        return {"reply": response.content}
