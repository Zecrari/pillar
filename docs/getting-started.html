🚀 Getting Started

This guide takes you from a fresh Python environment to a running Pillar API in under 5 minutes — with DI, background queue, health checks, and Swagger docs all working out of the box.

Installation #

Requirements

  • Python 3.10 or higher
  • pip (any recent version)

Install from PyPI

terminalbash
pip install pillar
💡 Tip — Rust engine

The pillar package ships with a pre-compiled Rust engine for Linux, macOS, and Windows (x86_64). If no compiled wheel is available for your platform, Pillar falls back to a pure-Python router automatically — everything still works.

For framework contributors (compile Rust from source)

terminalbash
# 1. Install Rust (https://rustup.rs)
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 2. Install maturin (Rust→Python build tool)
pip install maturin

# 3. Clone and compile
git clone https://github.com/Zecrari/pillar.git
cd pillar
maturin develop --release

# 4. Verify the Rust engine loaded
pillar info

Option A — Scaffold a Project #

The fastest way. One command creates a full domain skeleton with working users CRUD.

terminalbash
pillar new myapp
cd myapp
pip install -r requirements.txt
pillar run main:app --reload

Now open your browser:

  • http://localhost:8000/docs — Swagger UI
  • http://localhost:8000/redoc — ReDoc
  • http://localhost:8000/health — liveness probe
  • http://localhost:8000/guide — interactive route guide

Option B — Build Manually #

Walk through each layer so you understand exactly what Pillar does for you.

Step 1 · Repository #

The repository layer is the only place that touches the database. It may import from schemas but never from service or router.

domains/users/repository.pypython
from pillar.db import Database

class UserRepository:
    def __init__(self, db: Database):
        self.db = db
        self.db.execute("""
            CREATE TABLE IF NOT EXISTS users (
                id    INTEGER PRIMARY KEY AUTOINCREMENT,
                name  TEXT    NOT NULL,
                email TEXT    NOT NULL UNIQUE
            )
        """)

    def get_by_id(self, user_id: int) -> dict | None:
        return self.db.query(
            "SELECT * FROM users WHERE id = ?", (user_id,)
        )

    def get_all(self) -> list:
        return self.db.query_all("SELECT * FROM users")

    def create(self, name: str, email: str) -> dict:
        self.db.execute(
            "INSERT INTO users (name, email) VALUES (?, ?)", (name, email)
        )
        return self.get_by_id(self.db.last_insert_id())

Step 2 · Service #

The service layer contains all business logic. It may import from repository and schemas. UserRepository is injected automatically — no manual wiring.

domains/users/service.pypython
from pillar import background_task
from pillar.exceptions import NotFoundError, ConflictError
from .repository import UserRepository

class UserService:
    def __init__(self, repo: UserRepository):   # auto-injected
        self.repo = repo

    def get_user(self, user_id: int) -> dict:
        user = self.repo.get_by_id(user_id)
        if not user:
            raise NotFoundError(f"User {user_id} not found")
        return {**user, "status": "active"}

    def list_users(self) -> list:
        return [{**u, "status": "active"} for u in self.repo.get_all()]

    def create_user(self, name: str, email: str) -> dict:
        user = self.repo.create(name, email)
        self.send_welcome_email(user["email"])   # enqueued, non-blocking
        return {**user, "status": "active"}

    @background_task(retries=3)
    def send_welcome_email(self, email: str):
        # Runs in the Pillar Queue — no Celery, no Redis required
        print(f"[queue] Sending welcome email → {email}")

Step 3 · Schemas #

Pydantic models that can be shared across all layers.

domains/users/schemas.pypython
from pydantic import BaseModel, EmailStr, Field

class UserCreate(BaseModel):
    name:  str = Field(min_length=1, max_length=100)
    email: str

class UserResponse(BaseModel):
    id:     int
    name:   str
    email:  str
    status: str

Step 4 · Router #

HTTP definitions only. May import from service and schemas — never from repository directly. UserService is injected automatically.

domains/users/router.pypython
from pillar import Router
from .schemas import UserCreate, UserResponse
from .service import UserService

router = Router(prefix="/users", tags=["Users"])

@router.get("/", response_model=list[UserResponse], summary="List all users")
async def list_users(service: UserService):
    return service.list_users()

@router.get("/{user_id}", response_model=UserResponse, summary="Get a user")
async def get_user(user_id: int, service: UserService):
    return service.get_user(user_id)

@router.post("/", status_code=201, response_model=UserResponse,
             summary="Create a user")
async def create_user(data: UserCreate, service: UserService):
    return service.create_user(data.name, data.email)

@router.delete("/{user_id}", status_code=204, summary="Delete a user")
async def delete_user(user_id: int, service: UserService):
    service.delete_user(user_id)

Step 5 · Entry Point #

main.pypython
from pillar import Pillar
from domains.users.router import router as users_router

app = Pillar(
    title="My API",
    version="1.0.0",
    description="Built with Pillar — Production-Grade Python Backend",
)

app.include_router(users_router)

# Run: pillar run main:app --reload

Run the App #

terminalbash
# Development (hot-reload)
pillar run main:app --reload

# Production (multi-process)
pillar run main:app --workers 4 --host 0.0.0.0 --port 8000

⚠ Architecture is enforced

At startup, Pillar walks domains/ and parses every layer file with Python's AST. If router.py imports from repository.py, the server raises ArchitectureViolationError and refuses to start.

See the DDD Layer Enforcer section for the full import matrix.

Configuration #

Pillar reads pillar.toml from the working directory on startup. Every key has a sensible default — only add what you need to change.

pillar.tomltoml
[app]
title   = "My API"
version = "1.0.0"
debug   = false         # set true for stack traces in 500 responses
host    = "0.0.0.0"
port    = 8000

[database]
url       = "sqlite:///./app.db"
pool_size = 10

[queue]
driver        = "sqlite"    # "sqlite" | "redis" | "postgres"
db_path       = "pillar_queue.db"
workers       = 4
poll_interval = 0.5

[docs]
enabled = true              # false disables /docs, /redoc, /guide

[cors]
enabled       = true
allow_origins = ["*"]       # tighten for production

Environment variable overrides

HOST[app] host
PORT[app] port

REDIS_URL[queue] redis_url
